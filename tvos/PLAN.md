# audio_viz — tvOS App Plan

## Concept

A combined **music player + audio visualizer** for Apple TV. The app plays tracks from the user's Apple Music library and visualizes the audio in real time using the same Rust visualizer core as the terminal and web builds.

This approach is required because tvOS provides no API to capture audio from other apps (the Music app, etc.). By playing music through its own AVAudioEngine, the app can tap its own audio output for visualization.

## User Interface

### Screen 1 — Now Playing (primary / fullscreen)
- Fullscreen visualizer canvas (black background, cell-based rendering)
- Always-visible bottom control strip with gradient scrim
- Track info row: album art thumbnail (110×110 pt), song title/artist/album, progress bar
- Transport row: previous / play-pause / next buttons (64 pt spacing, hover effects)
- Action row: Library, Visualizers, Settings capsule buttons + active visualizer badge

### Screen 2 — Library Browser (sheet)
- Presented as a sheet from the action row
- Auth prompt shown if MusicKit not yet authorized
- Tabs: Songs / Albums / Playlists / Search (segmented picker)
- Albums tab: 4-column grid with 200×200 pt artwork cards
- Search tab: real-time async MusicCatalogSearchRequest (25 songs, 10 albums)
- Selecting any item starts playback and dismisses the sheet

### Screen 3 — Visualizer Picker (sheet)
- Horizontal LazyHStack carousel of visualizer cards (40 pt spacing)
- Each card: 240×150 pt preview tile with SF Symbol icon, name, and description
- Active visualizer shown with accent colour fill + checkmark badge
- Focus scale effect (1.05×) with easing animation
- Select to switch visualizer immediately via appState.switchVisualizer()

### Screen 4 — Settings Panel (sheet)
- Dynamic form driven by active visualizer's config JSON
- Float/Int fields: `+/−` button pairs (no Slider on tvOS), step = range/20
- Enum fields: horizontal ScrollView of inline buttons (not Picker — avoids blurry navigation sheet)
- Bool fields: standard Toggle
- Reset to Defaults button at bottom
- Changes applied immediately via bridge.setConfig()

## Rust Integration

### Build

The Rust library compiles to a static lib for tvOS using Rust's `aarch64-apple-tvos` target (Tier 3, requires nightly + `-Z build-std`):

```bash
cd tvos
./build-rust.sh           # release (default)
./build-rust.sh --debug   # debug build
```

The script builds for three targets:
- `aarch64-apple-tvos` (device) → `bridge/libaudio_viz.a`
- `aarch64-apple-tvos-sim` (arm64 simulator) → `bridge/libaudio_viz_sim.a`
- `x86_64-apple-tvos` (Intel simulator) → lipo'd into `libaudio_viz_sim.a`

### C FFI (audio_viz.h)

```c
// Lifecycle
AvizHandle aviz_create(const char* name, uint16_t cols, uint16_t rows);
void       aviz_destroy(AvizHandle handle);

// Per-frame
void        aviz_resize(AvizHandle handle, uint16_t cols, uint16_t rows);
void        aviz_tick(AvizHandle handle,
                      const float* fft, size_t fft_len,
                      const float* left, const float* right, size_t pcm_len,
                      float dt, uint32_t sample_rate);
const char* aviz_render(AvizHandle handle, float fps);

// Metadata
const char* aviz_name(AvizHandle handle);

// Config
const char* aviz_get_config(AvizHandle handle);
const char* aviz_set_config(AvizHandle handle, const char* json);

// Registry
const char* aviz_list_visualizers();
```

All `const char *` returns are owned by the handle and valid only until the next call to the same function on the same handle. Swift must copy immediately.

### Render Output

The Rust core renders ANSI-escaped terminal strings, then `parse_frame()` in `src/ffi.rs` converts them to a JSON cell array:

```json
[
  {"ch": "█", "col": 3, "row": 7, "r": 255, "g": 64, "b": 0, "bold": true, "dim": false},
  ...
]
```

`CellRenderer.swift` decodes this and draws each cell with CoreGraphics using a monospace font sized to fill the 80×45 grid on the 1920×1080 pt screen.

## Audio Pipeline

```
AVAudioEngine (AudioEngine.swift)
  └── AVAudioPlayerNode  ←── MusicPlayer schedules tracks here
        └── mainMixerNode
              └── installTap (4096-frame buffer)
                    ├── Sliding window → sampleBufferL / sampleBufferR
                    ├── vDSP Hann-windowed FFT → 2049 magnitude bins
                    └── onAudioFrame(fft, left, right, dt, sampleRate)
                          └── RustBridge.tick()  [audio thread]
```

Frame rate: 45 fps (matches FPS_TARGET). The tap fires at audio buffer boundaries; TimelineView drives rendering at display rate independently.

## MusicKit Integration

MusicKit (tvOS 17+) provides:
- `MusicLibrary` — access user's library (requires entitlement + user permission)
- `MusicCatalogSearch` — search Apple Music catalog
- `ApplicationMusicPlayer` — system music player (but uses its own audio session, not tappable)

**Important:** `ApplicationMusicPlayer` plays through the system, not through our AVAudioEngine — so we cannot tap it. The visualizer therefore animates on audio that flows through our engine (test tones, or future local-file support). When Apple Music is playing via ApplicationMusicPlayer, the visualizer shows an idle animation.
